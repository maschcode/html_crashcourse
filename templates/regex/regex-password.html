Special password regex description ( complex topic )

Regular Expression can be a very complex topic if u have more conditions to check. So it costs time to learn step by step complex group structures to make your own easy regex.

This example shows u how u can make an password regex to check that the user has minimum an low , big , number and special character in the string and this should be minium of 8 characters.

solution:

"^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$"


confused ? No Problem, lets look more on details:

  • "^...$" - this means u concentrate on special string which starts by ^ and ends by $ , the " - character stands only for string to use for pattern

  • (?=.*[a-z]) - ?= starts with condition and .* means that the point say every character and * to it say one or more of them, so minimum one character of small a-z

    (?=.*[A-Z]) - same meaning only for big alphabet characters
    (?=.*\d) - same meaning only for numbers of 0 - 9
    (?=.*[@$!%*?&]) - same meaning only for these special characters

  • [A-Za-z\d@$!%*?&]{8,} - this means that our string stands only with these characters and minimum 8 characters
